# Phase 2 — 구조 및 대표 코멘트 템플릿 재설계 (정의 고정본)

작성일: 2026-04-15  
적용 범위: `댓글 VoC 대시보드` + `영상 분석 요약` (공통 대표 코멘트 A 템플릿 포함)

## 0) 목적
이 문서는 화면을 “정보 나열”에서 “실무 판단”으로 전환하기 위한 Phase 2 구조 재설계 기준이다.  
핵심은 대표 코멘트 중심의 판단 흐름 고정, 유사 댓글 의미 정합성, 보조 지표의 역할 축소/정리다.

## 1) Phase 1 고정 정의 (반드시 준수)
| 항목 | 고정 정의 |
|---|---|
| 유사 댓글 성립 조건 | `같은 주장 방향 + 같은 문제 인식`을 동시에 만족해야 하며, 필터 범위 내 최소 3건 이상일 때만 유효 집합으로 간주 |
| 유사 댓글 미성립 처리 | 3건 미만 또는 의미 불일치 시 `근거 부족` 상태로 처리 (유사 댓글 숫자 과장 금지) |
| 대표 숫자 기준 | 대표 카드/expander/CSV 모두 `현재 필터 범위 기준` 동일 집계 사용 |
| cluster_size 사용 | `cluster_size`는 내부 참고값으로만 유지, 사용자 대표 지표로 사용 금지 |
| 대표 코멘트 선정 우선순위 | `반복성 > 주장 명확성 > 공감/확산 신호 > 최근성` (동등 가중치 금지) |
| 요약 길이 규칙 | 기본 1문장, 예외 시 최대 2문장 |

## 2) 대표 코멘트 템플릿 스키마 (최종)

### 2-1. Quick View (Top 3 카드)
```yaml
representative_quick_card:
  id: rep_id
  sentiment: positive|negative
  topic_label_ko: string
  summary_1line: string               # 잘린 원문 금지
  filter_scope_similar_count: int     # 현재 필터 기준
```
규칙: Quick View는 빠른 인지 전용이며, 대표성 설명/해석 지표(`왜 대표`, `issue_severity_level`)는 포함하지 않는다.

### 2-2. 상세 카드 읽기 순서 (고정)
```yaml
representative_detail_card:
  1_summary:
    user_intent: "이 이슈/강점을 1문장으로 이해"
    field: summary_1line_or_2line
  2_why_representative:
    user_intent: "왜 이 코멘트가 대표인지 납득"
    fields:
      - recurrence_evidence
      - resonance_evidence
      - context_relevance_evidence
      - issue_severity_level           # 관찰|주의|즉시관리 (상세 전용)
  3_original_comment:
    user_intent: "원문 검증"
    field: original_comment_full
  4_similar_comments:
    user_intent: "반복 신호 확인"
    fields:
      - filter_scope_similar_count
      - similarity_reason_labels_ko    # 예: 동일 불만 포인트 / 동일 강점 포인트 / 동일 대응 포인트
      - similar_comment_samples
```

### 2-3. 상세 카드 지표 블록 (분리 강제)
```yaml
metric_block:
  sentiment_intensity_pct: number   # 부정/긍정 강도 (원문 감정 강도)
  classification_confidence_pct: number  # 분류 확신도 (모델 확률)
  issue_strength_pct: number        # 운영 영향도 (반복성/확산/불편크기/대응필요성)
  issue_severity_level: 관찰|주의|즉시관리   # Quick View 미노출, 상세 카드 전용
  inquiry_flag: boolean
  numeric_question_flag: boolean
```

## 3) 필드별 역할 정의 (사용자 질문 중심)
| 필드 | 사용자 질문 | 허용 내용 | 금지 내용 |
|---|---|---|---|
| 요약 | “무슨 상황인가?” | 사용자 경험/문제상황 1문장 | 원문 잘라붙이기, 내부 로직 설명 |
| 왜 대표인가 | “왜 이게 중요한가?” | 반복 근거, 공감 근거, 맥락 근거 | 단순 점수 나열 |
| 원문 댓글 | “실제 표현이 맞나?” | 원문 전체 | 상단 중복 노출 |
| 유사 댓글 | “같은 대응이 필요한가?” | 동일 주장 방향+문제 인식 집합 | 주제만 같은 반대 의견 |
| 영상 맥락 인사이트 | “영상 맥락 대비 현실 괴리는?” | 영상 주장과 실사용 경험의 연결 해석 | NLP/가중치/메타 설명 |
| 해결 포인트 | “그래서 뭘 해야 하나?” | 댓글→문제→실행 방향의 자연어 제안 | 운영 템플릿 문구 반복 |

## 4) 기존 구조 vs 변경 구조
| 영역 | AS-IS | TO-BE |
|---|---|---|
| Quick View | 잘린 원문 + 일부 수치 | 1문장 요약 중심 단일 인지 구조 (대표 근거/이슈 레벨 미노출) |
| 상세 상단 | `왜 대표`와 `판단 근거` 중복 | 단일 `왜 대표 코멘트인가`로 통합 |
| 요약 | 원문 재진술/템플릿 문구 | 문제상황 요점 요약 (1~2문장) |
| 지표 | 부정강도=분류신뢰도 혼선 | 세 지표 분리(강도/신뢰/이슈강도) |
| 유사 댓글 수치 | 카드/expander/CSV 불일치 | 단일 집계 함수로 일치 |
| 유사 댓글 의미 | 주제 유사 중심 | 주장 방향 + 문제 인식 기준 |
| 영상 맥락 인사이트 | 내부 처리 과정 노출 | 실무 해석 결과만 노출 |
| 해결 포인트 | CX 운영 템플릿 | 기술/제품/운영 실행 가설 제시 |

## 5) 공통 템플릿 규칙 (VoC vs 영상요약)
공통 A 템플릿 필드는 동일 의미로만 사용한다.

| 구분 | 공통 필드 | 페이지 예외 |
|---|---|---|
| 공통 | 요약, 왜 대표, 원문, 유사 댓글, 핵심 지표, 키워드, 영상 맥락 인사이트, 해결 포인트 | 없음 |
| 영상 분석 요약 전용 | 영상 제목/채널/발행일/링크/타임스탬프 | 영상 탐색·선별 목적에서만 노출 |
| 댓글 VoC 전용 | 필터 범위 요약/대표군 빠른 카드 | 전체 여론 스냅샷 목적에서만 노출 |

규칙: 같은 필드명은 페이지가 달라도 의미가 바뀌면 안 된다.

## 6) 보조 지표/그래프 영역 재검증 결과 (유지/통합/제거 제안)
| 구성요소 | 제안 | 사유 |
|---|---|---|
| 주차별 긍정/부정 응답률 | 유지(격하) | 변화 감지용. 판단 트리거로 사용 금지 |
| 인사이트 유형 분포 | 통합 후보 | 반복 문의/대표 코멘트와 설명 중복 가능 |
| CEJ 단계별 고객 신뢰도 | 구조 변경 필요 | 단계 중심 가독성 부족. 단계 집계 우선으로 재구성 필요 |
| 브랜드 언급 비중 | 구조 변경 필요 | 브랜드 중심 해석이 어려움. 브랜드 축 우선 재배열 필요 |
| 반복 문의 코멘트 | 통합/재정의 | 개별 나열보다 “반복 주제 클러스터” 중심이 실무 적합 |
| 워드클라우드/도넛 | 유지(후순위) | 대표 코멘트 해석 보조로만 사용. 우선순위 신호 금지 |

## 7) 영상 분석 요약 페이지 구조 재설계
### 7-1. 영상 탐색 테이블(단일 테이블) 필수 컬럼
- 영상 제목
- 주요 언급 포인트(부정/긍정/혼합 가능)
- 전체 댓글 수
- 부정 밀도
- 긍정 밀도
- 반응 유형(혼합/부정 우세/긍정 우세)
- LG 연관성 여부
- LG 연관 근거(왜 연관인지)
- 언급 시점(가능 시)
- 영상 링크

`video_id`는 내부 식별자로만 유지하고 기본 UI 노출에서는 제외한다.

### 7-2. 부정 우세 영상 정의 (Phase 2 개념 고정)
부정 우세 영상은 단순 비율 차가 아니라 “실무 우선 관리 대상 부정 신호 누적”을 의미한다.  
Phase 2에서는 계산식 구현보다 개념 고정 우선:
- 반복성
- 확산성(공감/반응)
- 실사용 불편 규모
- 대응 필요성

## 8) 좌측 필터/실행 UX 재설계 요구사항
| 항목 | 재설계 기준 |
|---|---|
| 분석 결과 저장/불러오기 | 명시적 액션 버튼 제공 (`저장`, `불러오기`) + 마지막 선택 스냅샷 자동 복원 |
| 다중 검색 키워드 입력 | 쉼표 파싱 규칙을 UI에 명시. 권장: `ANY(OR)`/`ALL(AND)` 토글 제공 |
| 필터 오염 방지 | 검색 키워드와 제품 범위(product) 파이프라인 분리. 검색어가 product 후보로 유입되면 오류 처리 |

## 9) 선택 영상 전체 댓글 목록 개선 기준
- `LG 관련 여부`, `LG 관련 근거`, `문의 여부` 컬럼 추가
- `제품 타깃`, `분류 유형`, `신뢰도`는 한국어 표기
- `분류 이유`는 템플릿 문구 대신 간결한 판단 키워드/근거 중심
- 의미 불명확한 `맥락 필요` 컬럼은 제거 또는 역할 재정의
- `전체 댓글 목록` 우측에 엑셀 다운로드 버튼 제공

## 10) Phase 2 구현 전 체크리스트
- [ ] 대표 카드 읽기 순서가 `요약→근거→원문→유사댓글`로 고정됐는가
- [ ] 유사 댓글 수치가 카드/expander/CSV에서 100% 일치하는가
- [ ] 유사 댓글이 반대 의견을 포함하지 않는가
- [ ] 요약이 원문 절단/복붙이 아닌가
- [ ] 영상 맥락 인사이트에서 내부 처리 로직 문구가 제거됐는가
- [ ] 해결 포인트가 댓글 맥락에서 자연스럽게 이어지는가
- [ ] 보조 그래프가 대표 코멘트보다 앞에서 판단을 유도하지 않는가
- [ ] VoC/영상요약 페이지의 공통 템플릿 필드 의미가 일치하는가

## 11) 구현 단위 권장 순서 (참고)
1. 대표 코멘트 A 템플릿 공통 컴포넌트 정리  
2. 유사 댓글 집계 단일화 함수 고정  
3. 요약/근거 생성 포맷 분리  
4. 영상 맥락 인사이트·해결 포인트 문장 규약 적용  
5. 영상 탐색 테이블 컬럼 재배열 및 LG 연관성 필드 복원  
6. 필터 실행 UX 저장/불러오기 명시화  
7. 보조 지표 영역 통합/격하 적용
