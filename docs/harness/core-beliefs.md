# Design Docs — Core Beliefs

## 설계 철학

**"데이터가 주인공이다"**

이 대시보드는 꾸밈이 아니라 판단을 위한 도구다.
모든 UI 결정은 "이게 데이터를 더 잘 보이게 하는가?"
한 가지 기준으로 판단한다.

---

## 5가지 원칙

### 1. Clarity over Beauty
- 차트보다 숫자가 먼저
- 색상은 의미를 전달할 때만 사용
- 긍정 → 파란색(#4D79F6), 부정 → 주황색(#FFAA3B) 고정

### 2. Filter First
- 모든 화면은 필터(국가/브랜드/제품/기간) 기준으로 즉시 반응
- 필터 상태는 항상 화면 상단에 노출

### 3. Business Language
- 기술 용어 대신 업무 용어 사용
- "sentiment score" → "긍정 비율"
- "opinion_unit" → "의견 단위"
- 화면에 노출되는 모든 레이블은 한국어 우선

### 4. Export Ready
- 모든 주요 뷰는 추출 가능해야 함
- Raw Data → Excel (.xlsx)
- 임원 보고용 → PDF 출력 지원
- 버튼은 항상 우측 상단에 배치

### 5. Bilingual by Default
- UI 레이블: 한국어
- 원문 댓글: 원어 보존 + 한국어 번역 병기
- 영어 사용자도 필터/버튼을 이해할 수 있도록
  주요 액션 버튼은 한/영 병기 권장

## 컬러 시스템

| 용도 | 색상 | 코드 |
|------|------|------|
| 긍정 | 파란색 | `#4D79F6` |
| 부정 | 주황색 | `#FFAA3B` |
| 중립 | 회색 | `#A0AEC0` |
| 강조/액션 | 다크 블루 | `#1A56DB` |
| 배경 | 화이트 | `#FFFFFF` |

> 색상 임의 변경 금지. 긍/부정 색상은 전체 차트에서 일관되게 유지.

---

## 타이포그래피

- 한글 폰트: NanumGothic (Cloud), Malgun Gothic (로컬 fallback)
- 숫자/영문: Streamlit 기본 시스템 폰트
- KPI 카드 숫자: 크고 굵게 — 한눈에 읽혀야 함
- 본문/캡션: 작고 회색 — 보조 설명은 시선을 빼앗지 않을 것

---

## 레이아웃 원칙

```
[ 필터 바 — 항상 최상단 고정 ]

[ KPI 카드 행 — 3~5개 ]

[ 메인 차트 ]

[ 보조 테이블 / 키워드 패널 ]

[ 대표 댓글 ]

[ 다운로드 버튼 — 우측 상단 ]
```

- 카드/차트는 `st.container(border=True)`로 구분
- 한 화면에 스크롤 없이 KPI는 반드시 보여야 함
- 모바일(세로): 컬럼 → 단일 열로 자동 전환

---

## 출력 규칙

| 출력 유형 | 포함 내용 | 버튼 위치 |
|-----------|-----------|-----------|
| Raw Data Excel | 댓글 전체 + 필터 적용 상태 | 우측 상단 |
| PDF (임원용) | KPI 요약 + 주요 차트 + 대표 댓글 | 탭별 우측 상단 |

> PDF는 브라우저 인쇄 기능이 아닌
> 코드 기반 생성(예: `weasyprint` 또는 `reportlab`)으로 구현할 것.
> 한글 폰트 깨짐 방지를 위해 NanumGothic 임베딩 필수.
