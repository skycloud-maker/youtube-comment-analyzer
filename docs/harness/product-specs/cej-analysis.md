# product-specs/cej-analysis.md

## 기능 개요
고객경험여정(Customer Experience Journey) 단계별로
댓글 감성을 분류하여, 어느 단계에서 고객 불만/신뢰가
집중되는지 파악하는 분석 화면.

## 사용자 & 목적
- **제품 기획팀**: 구매 전/후 어느 단계에서 문제가 생기는지 파악
- **CS/AS팀**: 사용·관리·교체 단계 부정 댓글 집중 모니터링
- **경영진**: 전체 여정 중 브랜드 신뢰도 취약 구간 확인

---

## CEJ 단계 정의

| 순서 | 단계명 | 설명 | 예시 댓글 유형 |
|------|--------|------|----------------|
| 1 | 인지 | 제품/브랜드를 처음 접함 | "이런 제품이 있었군요" |
| 2 | 탐색 | 정보 검색, 비교 | "타사 제품이랑 비교하면?" |
| 3 | 결정 | 구매 의향, 고민 | "살까 말까 고민 중" |
| 4 | 구매 | 구매 완료 언급 | "방금 주문했어요" |
| 5 | 배송 | 배송 경험 | "배송이 너무 느려요" |
| 6 | 사용준비 | 설치·설정 | "설치가 복잡하네요" |
| 7 | 사용 | 실제 사용 경험 | "소음이 생각보다 크네요" |
| 8 | 관리 | 유지보수·AS | "AS 센터 연결이 안 돼요" |
| 9 | 교체 | 재구매·교체 고민 | "10년 썼는데 또 살게요" |
| 10 | 기타 | 위 단계 미해당 | - |

## 고객 신뢰도 지수 계산

```
신뢰도 지수 = ((긍정 수 − 부정 수) ÷ (긍정 수 + 부정 수)) × 50 + 50
```

- 범위: 0 ~ 100
- 50 = 긍정/부정 동일
- 100 = 전체 긍정
- 0 = 전체 부정

> 긍정+부정 = 0인 경우 신뢰도 지수 = 50 (중립) 처리

## 화면 명세

### CEJ 단계별 고객 신뢰도 테이블

| 표시 컬럼 | 설명 |
|-----------|------|
| 고객여정단계 | 인지~기타 (CEJ_ORDER 순 정렬) |
| product | 제품명 |
| 국가 | KR / US |
| 브랜드 | LG / Samsung 등 |
| 댓글 수 | 해당 단계 전체 댓글 수 |
| 긍정 댓글 수 | sentiment = positive |
| 부정 댓글 수 | sentiment = negative |
| 고객 신뢰도 지수 | 위 공식 적용값 |

- 필터(국가/브랜드/제품/기간) 연동
- 데이터 없을 시: "현재 조건에서 표시할 CEJ 데이터가 없습니다"

### CEJ 부정 비율 차트 (추가 구현 목표)
- X축: CEJ 단계 (인지 → 교체 순)
- Y축: 부정 비율 (%)
- 색상: 부정 비율 높을수록 진한 주황 (`#FFAA3B` 계열)
- 목적: 어느 단계가 가장 취약한지 한눈에 파악

### CEJ × 제품 히트맵 (추가 구현 목표)
- 행: CEJ 단계
- 열: 제품 카테고리
- 셀값: 신뢰도 지수
- 목적: "세탁기 + 배송 단계" 같은 교차 취약점 발견
