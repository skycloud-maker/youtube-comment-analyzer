# product-specs/data-pipeline-automation.md

## 기능 개요
YouTube Data API를 통해 지정된 채널의 영상·댓글 데이터를
자동으로 수집·전처리·저장하는 파이프라인.
현재 수동 실행 구조를 트리거 기반 자동화로 전환하는 것이 목표.

## 목표 동작
- 사용자가 대시보드에서 "수집 실행" 버튼을 누르면 즉시 수집 시작
- 수집 완료 후 parquet 저장 → 캐시 자동 갱신 → 대시보드 즉시 반영
- 수집 진행 상태를 화면에서 실시간으로 확인 가능

---

## 수집 대상

| 구분 | 채널 | 비고 |
|------|------|------|
| 자사 | 2~3개 브랜드 채널 | 환경변수로 채널 ID 관리 |
| 국내 경쟁사 | LG, Samsung | |
| 해외 경쟁사 | GE, Whirlpool 등 | 다국어 댓글 포함 |

## 수집 범위 (1회 실행 기준)
- 채널당 최신 영상 N개 (설정값, 기본 50개)
- 영상당 댓글 전체 (페이지네이션 처리)
- 이미 수집된 댓글은 중복 제외 (video_id + comment_id 기준)

---

## 파이프라인 단계

```
1. 트리거
   대시보드 "수집 실행" 버튼 클릭
           ↓
2. 채널 목록 로드
   환경변수 YOUTUBE_CHANNEL_IDS 파싱
           ↓
3. YouTube Data API 호출
   - 채널별 영상 목록 수집
   - 영상별 댓글 수집 (CommentThreads.list)
   - API quota 초과 시 수집 중단 + 진행 상태 저장
           ↓
4. 전처리
   - 언어 감지
   - 한국어 번역 (해외 댓글)
   - 스팸/노이즈 제거
   - 감성 분류 (positive/negative/neutral/excluded)
   - CEJ 단계 태깅
   - 제품 분류
   - 의견 단위(opinion_unit) 분리
           ↓
5. 저장
   data/processed/{run_id}/ 하위 parquet 14종 저장
           ↓
6. 캐시 갱신
   CACHE_VERSION 자동 업데이트
   dashboard_bundle.pkl 재생성
           ↓
7. 대시보드 반영
   st.cache_data 무효화 → 자동 리로드
```

## UI 명세 (대시보드 내 수집 패널)

### 수집 실행 버튼
- 위치: 사이드바 또는 설정 탭
- 버튼명: "댓글 수집 실행"
- 클릭 시: 진행 상태 바 + 단계별 로그 표시
  ```
  ✅ 채널 목록 로드 완료 (6개)
  ⏳ LG 채널 영상 수집 중... (23/50)
  ✅ Samsung 채널 완료 (1,243개 댓글)
  ...
  ```
- 완료 시: "수집 완료 — 총 N개 댓글 수집됨" 메시지 + 자동 새로고침

### 수집 이력
- 마지막 수집 일시 표시
- 회차별 수집 건수 간단 요약

## 환경변수 명세

| 변수명 | 설명 | 예시 |
|--------|------|------|
| YOUTUBE_API_KEY | YouTube Data API 키 | `AIza...` |
| YOUTUBE_CHANNEL_IDS | 수집 대상 채널 ID (쉼표 구분) | `UCxxx,UCyyy` |
| MAX_VIDEOS_PER_CHANNEL | 채널당 최대 영상 수 | `50` |

> 모든 값은 Streamlit Cloud Secrets 또는 .env 파일로 관리
> 코드 하드코딩 절대 금지 (AGENTS.md 참조)

## 제약 사항 및 주의

### YouTube API Quota
- 기본 할당량: 하루 10,000 유닛
- 댓글 수집 1회 = 영상당 약 1~5 유닛 소비
- 채널 6개 × 50영상 기준 약 300~1,500 유닛 예상
- Quota 초과 시: 수집 중단 + 완료된 부분까지만 저장

### Streamlit Cloud 제약
- 장시간 실행(10분+) 시 타임아웃 위험
- 대용량 수집은 백그라운드 실행 불가
- **현실적 권장 방식**: 채널당 최신 영상 20~30개로 제한하거나
  수집을 청크 단위로 나눠서 실행

### 향후 서버 이전 시
- GitHub Actions 스케줄러로 야간 자동 실행 전환 권장
- 수집 결과를 레포가 아닌 외부 스토리지(S3, GCS)에 저장
- 이 스펙의 파이프라인 단계는 서버 환경과 무관하게 유지
